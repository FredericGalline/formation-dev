# Thème 02 — Outils du développeur

> **Le but :** installer et maîtriser l'**atelier** du développeur. Avant d'écrire du
> code « pour de vrai », il faut savoir manier son éditeur, son terminal, et surtout
> **Git** — l'outil qu'on utilise tous les jours, toute sa carrière.

## Pourquoi ce thème maintenant

On a le modèle mental du web (Thème 01). Avant d'attaquer HTML/CSS, on se dote des
outils qui serviront dans **tous** les thèmes suivants. Un développeur qui ne maîtrise
pas Git ou son terminal perd un temps fou et finit par avoir peur de casser des
choses. Ici, on enlève cette peur.

## Ce que l'apprenant saura faire à la fin

- Travailler dans VSCode : ouvrir un projet, naviguer vite, utiliser le terminal intégré.
- Se déplacer et manipuler des fichiers en **ligne de commande**.
- **Versionner** son travail avec Git (commits propres, historique).
- Collaborer via **GitHub/GitLab** : branches, push/pull, Pull Request / Merge Request, conflit.
- Connaître les **workflows Git** (stratégies de branches) et les **conventions** (commits, nommage, tags).
- Comprendre ce qu'est une **dépendance** et installer un projet avec npm / composer.
