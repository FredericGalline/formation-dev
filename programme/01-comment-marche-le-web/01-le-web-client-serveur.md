# 01.1 — Le web : client, serveur, requête/réponse

> **Fiche-référentiel.** Elle dit au tuteur *quoi* enseigner et *comment juger la
> maîtrise*. La pédagogie (exercices, QCM, analogies) est improvisée par l'IA, à la
> portée de l'apprenant.

- **Thème :** 01 — Comment marche le web
- **Prérequis :** aucun
- **Idée centrale :** le web est un **dialogue** entre une machine qui demande et une
  machine qui répond.

## 🎯 Objectifs

À la fin, l'apprenant comprend que naviguer sur le web, c'est échanger des
requêtes et des réponses entre un **client** et un **serveur**.

## 📚 Notions à connaître

- 🎯 **Web ≠ Internet.** Internet = le réseau (les tuyaux) ; le web = un service qui
  circule dessus (les pages). L'e-mail aussi circule sur Internet, mais ce n'est pas du web.
- 🎯 **Le modèle client / serveur.** Le **client** demande (ton navigateur), le
  **serveur** répond (une machine quelque part qui détient le site).
- 🎯 **Requête / réponse.** Toute page affichée est le résultat d'une *requête* envoyée
  et d'une *réponse* reçue. Rien n'apparaît « tout seul ».
- 🎯 **Les rôles concrets.** Navigateur = client. Serveur web = programme qui écoute les
  requêtes et renvoie des fichiers (HTML, images…) ou du contenu généré.
- 📌 **Adresse IP, nom de domaine, DNS.** Une machine a une adresse (IP) ; le nom de
  domaine (`exemple.fr`) est plus lisible ; le **DNS** est l'annuaire qui traduit l'un
  en l'autre.
- 📖 *(culture)* **Où « vit » un site.** Hébergement, data center : le serveur est une
  vraie machine allumée quelque part, pas une abstraction.

## ✅ Critères de maîtrise

L'apprenant valide quand il sait, **sans aide** :

1. Expliquer avec ses mots le trajet : *« je tape une adresse → mon navigateur (client)
   envoie une requête → un serveur répond → la page s'affiche »*.
2. Sur un exemple donné (consulter un blog, envoyer un message), **désigner qui est le
   client et qui est le serveur**.
3. Distinguer **web** et **Internet** en une phrase juste.

## ⚠️ Pièges courants

- Confondre **web** et **Internet**.
- Croire que « le site est dans le navigateur » (le site vit sur le serveur ; le
  navigateur en reçoit une copie le temps de l'afficher).
- Penser que le serveur est forcément un seul gros ordinateur unique.
